# Sketch 003: Header / Receipt / Error Frame

## Design Question

文字角色都定了(B + B + lavender)。剩下最后一个结构题:**高光时刻怎么收尾?**

- setup wizard 跑完那一刻
- `slate clean` 完成那一刻
- 出错那一刻

3 档:不画 / 树枝 / 卡片。

## How to View

```
open .planning/sketches/003-header-receipt/index.html
```

## Variants

- **A: Bare** — 没有框,所有高光时刻都是一行 plain text + 粉 + ★。最 Unix、最快。
- **B: Tree** — ◆ 标题 + ┃ 树枝 + └ 收尾。charm.sh 经典叙事感。
- **C: Card** — 重要时刻有 `╭─╮` 画框(setup 完成 / error)。日常命令仍保持 inline。剧场感最强。

## What to Look For

- 场景 1 是 30 秒 install 后的"wow"时刻——你希望它有多隆重?
- 场景 4 error 行:框框是帮你聚焦还是占地方?
- 同一个 `slate theme` 重复跑很多次时,B / C 哪种不会显得过度?
